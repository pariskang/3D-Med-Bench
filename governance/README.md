# 数据治理与合规

## 适用法规

| 法规/标准 | 适用场景 |
|---|---|
| 《个人信息保护法》(PIPL) | 敏感个人信息（健康信息）处理需单独同意 |
| 《数据安全法》 | 重要数据（≥100万人或≥10万敏感个人信息）需安全评估 |
| GB/T 42460-2023 | 去标识化效果评估，直接/准标识符清除 |
| GB/T 39725-2020 | 健康医疗数据六类五级分类分级 |
| GB/T 45652-2025 | 生成式AI训练数据安全，健康数据严格去标识 |
| GB 45438-2025 | AIGC标识，病人化身须标注"AI生成" |
| 《卫健委数据分类分级指南（试行）》2024 | 判断是否触发重要数据阈值 |

## 目录结构

```
governance/
├── deid_report/         ← 去标识化效果评估报告（每批次）
├── ethics_approval/     ← 伦理委员会审批文件
└── data_class_level/    ← 数据分类分级定级文件
```

## 合规落地要点

1. **去标识**：Stage 2 自动执行，产出 deid_report/{case_id}_deid.json
2. **分级**：每个案例标注 data_class_level（一~五级）
3. **不出域**：2万例本体保留院内，发布仅开源代码+合成样例
4. **AIGC标识**：化身视频帧嵌入 C2PA 元数据 + 显式水印
5. **伦理**：按回顾性真实世界研究申报，涉基因另行申请
